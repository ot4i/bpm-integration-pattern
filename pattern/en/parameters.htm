Parameters for the BPM Integration Service pattern

Pattern name
Pattern parameter M C Default Description
Pattern Instance Name Yes No  

The unique name used to identify all artifacts generated by the pattern instance under the pattern instance project.
BPM
Pattern parameter M C Default Description
BPM Integration service Yes Yes  

The BPM Integration Service to be implemented by the Integration Bus service.
Service information
Pattern parameter M C Default Description
Integration Bus Service name Yes Yes  

A wizard dialog to create a new service or use existing service to implement the BPM Integration Service.

All the pattern parameters are listed with their default values.

The column headed M indicates whether the property is mandatory. The column headed C indicates whether the property is configurable, meaning that you can change the value when you add the message flow to the IBM Integration Bus archive (BAR) file.

Where no default is indicated the default value is an empty string.
