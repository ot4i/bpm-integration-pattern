Log subflow for the Data distribution SAP to WebSphere MQ: one-way (for IDoc) pattern

If the Log subflow is included in the IDoc Routing flow, each successful message is logged. The logging of requests is controlled by the user-defined property: LoggingOn.

If this property is selected, logging is required, the Log subflow is called and ends the flow. If the logging fails, the exception is caught within the flow and the information is written to a Trace node.

Log messages are written as persistent WebSphere MQ messages to a WebSphere MQ queue. These log messages are then processed separately in accordance with the standard practices within your organization.

The message log contains the original message and the following information in an MQRFH2 header:
  • Time stamp
  • Broker name
  • Message flow name
