Error subflow for the Data distribution SAP to WebSphere MQ: one-way (for IDoc) pattern

If errors are caught while processing the IDoc Routing message flow, the Error subflow is called.

If you configure the pattern instance to enable error messages, a message is written to the error queue. The following actions are taken by the Error subflow:

  • If the Error subflow is in the IDoc Routing flow, the message that is written to the error queue is a standard XML message, as shown by the following example: 
    <Error><Status>-1</Status><Text>Failed to send message</Text></Error>
  • The exception list is added to the message.
  • If you have configured the pattern parameters to include error messages, the code to write error messages is included in the Error subflow.
  • Error logging is controlled by the user-defined property: ErrorLoggingOn. If this property is selected, an error message is written to the error queue.

    The error message contains the broker name, flow name, and a time stamp. The exception list is summarized, and the bitstream content of the message that is causing the error is added as a BLOB.

If an error occurs, the Error flow produces the exception again so that the SAPInput node rolls back the transaction in SAP.

Messages are backed out if error messages are not selected in the pattern configuration.
